Las APIs web (Interfaz de Programación de Aplicaciones) son componentes fundamentales en el desarrollo de aplicaciones modernas, permitiendo la comunicación entre diferentes sistemas. En el ecosistema Python, Flask y FastAPI se han posicionado como dos de los frameworks más populares para crear APIs RESTful. Por otro lado, Node.js es ampliamente utilizado para consumir estas APIs desde aplicaciones JavaScript del lado del servidor o cliente.

Introducción

En el desarrollo de aplicaciones modernas, es común utilizar diferentes tecnologías para el backend y frontend. Python, con frameworks como Flask y FastAPI, se ha consolidado como una excelente opción para crear APIs robustas y eficientes. Por otro lado, Node.js ofrece un entorno ideal para construir aplicaciones que consumen estas APIs, especialmente cuando se trabaja con JavaScript en todo el stack.

La comunicación entre estos sistemas se realiza principalmente a través de APIs RESTful, que utilizan el protocolo HTTP y formatos de datos como JSON para intercambiar información. Este artículo explora en detalle cómo crear APIs con Flask y FastAPI, y cómo consumirlas desde aplicaciones Node.js.

Flask

Flask es un microframework para Python basado en Werkzeug y Jinja2. Es minimalista, flexible y fácil de usar, lo que lo hace ideal para proyectos pequeños y medianos, así como para prototipado rápido.

Características principales

  • Minimalista: Proporciona solo lo esencial, permitiendo al desarrollador añadir extensiones según sus necesidades.
  • Flexible: No impone una estructura específica de proyecto.
  • Extensible: Cuenta con un ecosistema de extensiones para funcionalidades adicionales.
  • Documentación completa: Amplia documentación y comunidad activa.
  • Compatibilidad: Funciona con cualquier base de datos y sistema de plantillas.

Ejemplo básico

A continuación, un ejemplo básico de una API REST con Flask:

from flask import Flask, jsonify, request

app = Flask(__name__)

# Datos de ejemplo
users = [
    {"id": 1, "name": "Juan Pérez", "email": "juan@example.com"},
    {"id": 2, "name": "María García", "email": "maria@example.com"}
]

@app.route('/users', methods=['GET'])
def get_users():
    return jsonify(users)

@app.route('/users/', methods=['GET'])
def get_user(user_id):
    user = next((u for u in users if u['id'] == user_id), None)
    if user:
        return jsonify(user)
    return jsonify({"error": "Usuario no encontrado"}), 404

@app.route('/users', methods=['POST'])
def create_user():
    data = request.get_json()
    new_user = {
        "id": len(users) + 1,
        "name": data.get('name'),
        "email": data.get('email')
    }
    users.append(new_user)
    return jsonify(new_user), 201

if __name__ == '__main__':
    app.run(debug=True)

FastAPI

FastAPI es un framework moderno y rápido para construir APIs con Python 3.6+ basado en type hints estándar de Python. Se caracteriza por su alto rendimiento, comparable con Node.js y Go.

Características principales

  • Alto rendimiento: Uno de los frameworks más rápidos disponibles, gracias a Starlette y Pydantic.
  • Desarrollo rápido: Reduce en aproximadamente un 40% los errores humanos gracias al sistema de tipos.
  • Documentación automática: Genera automáticamente documentación interactiva (Swagger UI y ReDoc).
  • Basado en estándares: Compatible completamente con los estándares OpenAPI y JSON Schema.
  • Soporte nativo para async/await: Permite escribir código asíncrono de forma natural.

Ejemplo básico

El mismo ejemplo anterior implementado con FastAPI:

from fastapi import FastAPI, HTTPException
from pydantic import BaseModel
from typing import List

app = FastAPI()

# Modelo de datos
class User(BaseModel):
    id: int
    name: str
    email: str

class UserCreate(BaseModel):
    name: str
    email: str

# Datos de ejemplo
users = [
    User(id=1, name="Juan Pérez", email="juan@example.com"),
    User(id=2, name="María García", email="maria@example.com")
]

@app.get("/users", response_model=List[User])
async def get_users():
    return users

@app.get("/users/{user_id}", response_model=User)
async def get_user(user_id: int):
    user = next((u for u in users if u.id == user_id), None)
    if not user:
        raise HTTPException(status_code=404, detail="Usuario no encontrado")
    return user

@app.post("/users", response_model=User)
async def create_user(user: UserCreate):
    new_user = User(
        id=len(users) + 1,
        name=user.name,
        email=user.email
    )
    users.append(new_user)
    return new_user

Comparación entre Flask y FastAPI

Aunque ambos frameworks permiten crear APIs en Python, presentan diferencias importantes:

  • Rendimiento: FastAPI es significativamente más rápido que Flask, especialmente en operaciones I/O.
  • Validación de datos: FastAPI valida automáticamente los tipos de datos usando Pydantic, mientras que en Flask se requiere código adicional o extensiones.
  • Documentación: FastAPI genera documentación automática, Flask requiere herramientas externas.
  • Asincronía: FastAPI soporta nativamente async/await, Flask requiere extensiones como Flask-Asyncio.
  • Curva de aprendizaje: Flask es más simple para principiantes, FastAPI requiere conocimiento de type hints y conceptos más avanzados.
  • Comunidad y ecosistema: Flask tiene una comunidad más grande y más extensiones disponibles.

Node.js

Node.js es un entorno de ejecución para JavaScript construido con el motor V8 de Chrome. Permite ejecutar código JavaScript en el servidor, lo que facilita el desarrollo de aplicaciones full-stack con un único lenguaje.

Características principales

  • Asíncrono y orientado a eventos: Maneja operaciones I/O de forma no bloqueante.
  • Un solo hilo: Utiliza un modelo de bucle de eventos para manejar múltiples conexiones concurrentes.
  • NPM: Gestor de paquetes más grande del ecosistema JavaScript.
  • Alto rendimiento: Ideal para aplicaciones que requieren alta concurrencia.
  • Universal JavaScript: Permite usar el mismo lenguaje en frontend y backend.

Consumo de APIs desde Node.js

Node.js ofrece varias formas de consumir APIs externas. Las dos opciones más comunes son Axios (biblioteca de terceros) y Fetch API

Uso de Axios

Axios es una popular biblioteca para realizar peticiones HTTP. Ofrece una API sencilla y características avanzadas como interceptores y cancelación de peticiones.

const axios = require('axios');

// Obtener todos los usuarios
async function getUsers() {
    try {
        const response = await axios.get('http://localhost:8000/users');
        console.log(response.data);
    } catch (error) {
        console.error('Error al obtener usuarios:', error.message);
    }
}

// Crear un nuevo usuario
async function createUser(userData) {
    try {
        const response = await axios.post('http://localhost:8000/users', userData);
        console.log('Usuario creado:', response.data);
    } catch (error) {
        console.error('Error al crear usuario:', error.message);
    }
}

// Ejemplo de uso
getUsers();
createUser({ name: "Carlos López", email: "carlos@example.com" });

Uso de Fetch

Desde Node.js 18, la API Fetch está disponible de forma nativa, eliminando la necesidad de bibliotecas externas para casos simples.

// Obtener todos los usuarios
async function getUsers() {
    try {
        const response = await fetch('http://localhost:8000/users');
        if (!response.ok) {
            throw new Error(`Error HTTP: ${response.status}`);
        }
        const users = await response.json();
        console.log(users);
    } catch (error) {
        console.error('Error al obtener usuarios:', error.message);
    }
}

// Crear un nuevo usuario
async function createUser(userData) {
    try {
        const response = await fetch('http://localhost:8000/users', {
            method: 'POST',
            headers: {
                'Content-Type': 'application/json',
            },
            body: JSON.stringify(userData),
        });
        
        if (!response.ok) {
            throw new Error(`Error HTTP: ${response.status}`);
        }
        
        const newUser = await response.json();
        console.log('Usuario creado:', newUser);
    } catch (error) {
        console.error('Error al crear usuario:', error.message);
    }
}

// Ejemplo de uso
getUsers();
createUser({ name: "Ana Martínez", email: "ana@example.com" });

Ejemplos prácticos

API completa con Flask

Un ejemplo más completo de API con Flask que incluye autenticación JWT y base de datos:

from flask import Flask, jsonify, request
from flask_jwt_extended import JWTManager, create_access_token, jwt_required
from flask_sqlalchemy import SQLAlchemy
from werkzeug.security import generate_password_hash, check_password_hash

app = Flask(__name__)
app.config['SQLALCHEMY_DATABASE_URI'] = 'sqlite:///users.db'
app.config['JWT_SECRET_KEY'] = 'super-secret-key'

db = SQLAlchemy(app)
jwt = JWTManager(app)

class User(db.Model):
    id = db.Column(db.Integer, primary_key=True)
    username = db.Column(db.String(80), unique=True, nullable=False)
    password_hash = db.Column(db.String(120), nullable=False)

@app.route('/login', methods=['POST'])
def login():
    username = request.json.get('username')
    password = request.json.get('password')
    
    user = User.query.filter_by(username=username).first()
    
    if user and check_password_hash(user.password_hash, password):
        access_token = create_access_token(identity=username)
        return jsonify(access_token=access_token)
    
    return jsonify({"msg": "Credenciales incorrectas"}), 401

@app.route('/protected', methods=['GET'])
@jwt_required()
def protected():
    return jsonify(msg="Acceso concedido a ruta protegida")

if __name__ == '__main__':
    with app.app_context():
        db.create_all()
    app.run(debug=True)

API completa con FastAPI

El mismo ejemplo implementado con FastAPI:

from fastapi import FastAPI, Depends, HTTPException, status
from fastapi.security import HTTPBearer, HTTPAuthorizationCredentials
from pydantic import BaseModel
from sqlalchemy import create_engine, Column, Integer, String
from sqlalchemy.ext.declarative import declarative_base
from sqlalchemy.orm import sessionmaker, Session
import jwt
from passlib.context import CryptContext

app = FastAPI()
security = HTTPBearer()
pwd_context = CryptContext(schemes=["bcrypt"], deprecated="auto")

# Configuración de base de datos
SQLALCHEMY_DATABASE_URL = "sqlite:///./users.db"
engine = create_engine(SQLALCHEMY_DATABASE_URL)
SessionLocal = sessionmaker(autocommit=False, autoflush=False, bind=engine)
Base = declarative_base()

class User(Base):
    __tablename__ = "users"
    id = Column(Integer, primary_key=True, index=True)
    username = Column(String, unique=True, index=True)
    password_hash = Column(String)

Base.metadata.create_all(bind=engine)

# Modelos
class UserCreate(BaseModel):
    username: str
    password: str

class Token(BaseModel):
    access_token: str
    token_type: str

# Dependencias
def get_db():
    db = SessionLocal()
    try:
        yield db
    finally:
        db.close()

def verify_password(plain_password, hashed_password):
    return pwd_context.verify(plain_password, hashed_password)

def get_password_hash(password):
    return pwd_context.hash(password)

def create_access_token(data: dict):
    return jwt.encode(data, "super-secret-key", algorithm="HS256")

@app.post("/register")
def register(user: UserCreate, db: Session = Depends(get_db)):
    db_user = db.query(User).filter(User.username == user.username).first()
    if db_user:
        raise HTTPException(status_code=400, detail="Usuario ya existe")
    
    hashed_password = get_password_hash(user.password)
    new_user = User(username=user.username, password_hash=hashed_password)
    db.add(new_user)
    db.commit()
    db.refresh(new_user)
    return {"msg": "Usuario registrado exitosamente"}

@app.post("/login", response_model=Token)
def login(user: UserCreate, db: Session = Depends(get_db)):
    db_user = db.query(User).filter(User.username == user.username).first()
    if not db_user or not verify_password(user.password, db_user.password_hash):
        raise HTTPException(
            status_code=status.HTTP_401_UNAUTHORIZED,
            detail="Credenciales incorrectas"
        )
    
    access_token = create_access_token(data={"sub": user.username})
    return {"access_token": access_token, "token_type": "bearer"}

@app.get("/protected")
def protected(credentials: HTTPAuthorizationCredentials = Depends(security)):
    try:
        payload = jwt.decode(credentials.credentials, "super-secret-key", algorithms=["HS256"])
        username = payload.get("sub")
        return {"msg": f"Acceso concedido a {username}"}
    except jwt.PyJWTError:
        raise HTTPException(
            status_code=status.HTTP_401_UNAUTHORIZED,
            detail="Token inválido"
        )

Consumo desde Node.js

Ejemplo de consumo de la API protegida desde Node.js:

const axios = require('axios');

class APIClient {
    constructor(baseURL) {
        this.baseURL = baseURL;
        this.token = null;
    }
    
    async login(username, password) {
        try {
            const response = await axios.post(`${this.baseURL}/login`, {
                username,
                password
            });
            
            this.token = response.data.access_token;
            console.log('Login exitoso');
            return true;
        } catch (error) {
            console.error('Error en login:', error.response?.data?.detail || error.message);
            return false;
        }
    }
    
    async getProtectedData() {
        if (!this.token) {
            console.error('No hay token de autenticación');
            return null;
        }
        
        try {
            const response = await axios.get(`${this.baseURL}/protected`, {
                headers: {
                    'Authorization': `Bearer ${this.token}`
                }
            });
            
            return response.data;
        } catch (error) {
            console.error('Error al obtener datos protegidos:', error.response?.data?.detail || error.message);
            return null;
        }
    }
}

// Uso del cliente
async function main() {
    const client = new APIClient('http://localhost:8000');
    
    // Iniciar sesión
    const loggedIn = await client.login('usuario', 'contraseña');
    
    if (loggedIn) {
        // Acceder a ruta protegida
        const protectedData = await client.getProtectedData();
        console.log('Datos protegidos:', protectedData);
    }
}

main();

Mejores prácticas

Para APIs en Flask/FastAPI:

  • Validación de datos: Siempre validar y sanitizar los datos de entrada.
  • Manejo de errores: Implementar un manejo consistente de errores con códigos HTTP apropiados.
  • Autenticación y autorización: Usar JWT u OAuth2 para proteger endpoints sensibles.
  • Documentación: Documentar la API usando OpenAPI/Swagger.
  • Rate limiting: Implementar límites de tasa para prevenir abusos.
  • Logging: Registrar actividades importantes para monitoreo y debugging.
  • Pruebas: Escribir pruebas unitarias e integrales para la API.

Para consumo desde Node.js:

  • Manejo de errores: Implementar un manejo robusto de errores en las peticiones.
  • Timeouts: Configurar timeouts apropiados para evitar peticiones bloqueantes.
  • Retry logic: Implementar lógica de reintento para peticiones fallidas.
  • Circuit breaker: Usar patrones como circuit breaker para prevenir fallos en cascada.
  • Seguridad: Validar respuestas y sanitizar datos recibidos de la API.
  • Cache: Implementar cache cuando sea apropiado para mejorar el rendimiento.
